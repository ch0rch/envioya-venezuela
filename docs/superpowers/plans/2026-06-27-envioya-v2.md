```ts
export function availableCorridors(ds: Dataset): Corridor[] {
  const byCurrency = new Map<string, string>(); // currency1 -> system1
  for (const r of ds.rates) {
    if (!byCurrency.has(r.currency1)) byCurrency.set(r.currency1, r.system1);
  }
  return [...byCurrency.entries()]
    .map(([currency, systemId]) => ({
      currency,
      systemId,
      label: CORRIDOR_LABELS[currency]?.label ?? currency,
      flag: CORRIDOR_LABELS[currency]?.flag ?? '💱',
    }))
    .sort((a, b) => a.label.localeCompare(b.label, 'es'));
}
```

- [ ] **Step 1: Empirically verify the `/a/` amount semantics FIRST (this determines the format the test pins)**

The `/a/` route format is `/{locale}/a/{system1}/{system2}/{amount1}/{amount2}` (locale `es-VE`). The send-vs-receive assignment of the two amounts is NOT certain from static reading. Verify against the real site:

```bash

# candidate B: amount on the second slot
curl -s -o /dev/null -w "B %{http_code}\n" "https://saldo.com.ar/es-VE/a/zinli/pago_movil/0/100"
```
Both likely return 200 (SSR page). To decide which pre-fills the SEND amount of 100 from zinli, fetch the HTML and look for the value 100 bound to the origin (zinli/USD) input vs the destination (pago_movil/VES) input:
```bash
curl -s "https://saldo.com.ar/es-VE/a/zinli/pago_movil/100/0" | grep -oE 'value="100"|zinli|pago_movil' | head
```
Pick the candidate where the origin (zinli) amount is 100. **Record the chosen format in the report.** If neither can be confirmed (page is JS-rendered and the value isn't in the HTML, or behavior is ambiguous), use the `/cobrar/` fallback as the PRIMARY output and note it.

- [ ] **Step 4: Implement** `src/lib/saldoar-link.ts` (use the format confirmed in Step 1):

```ts
const LOCALE = 'es-VE';
const BASE = 'https://saldo.com.ar';
const DEST = 'pago_movil';

export const COBRAR_FALLBACK_BASE = `${BASE}/${LOCALE}/cobrar`;

// `/a/{system1}/{system2}/{amount1}/{amount2}` carries the amount.
// Format confirmed empirically (see plan Task 4 Step 1).
export function buildSaldoarLink(systemId: string, amount: number): string {
  const amt = Math.max(0, Math.round(amount));
  return `${BASE}/${LOCALE}/a/${systemId}/${DEST}/${amt}/0`;
}

// Fallback when amount semantics are unconfirmed: par only, no amount.
export function buildCobrarLink(systemId: string): string {
  return `${COBRAR_FALLBACK_BASE}/${systemId}-a-${DEST}`;
}
```
